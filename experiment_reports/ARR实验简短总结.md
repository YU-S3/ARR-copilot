# ARR 三分类实验简短总结

## 1. 问题定义

本项目基于 ARR 临床真实世界表格数据，构建三分类诊断模型，目标列为`确诊（0为排除；1为确诊；2为灰色区域）`。任务目标是在有限样本和明显类别不均衡条件下，尽量同时兼顾总体判别能力与少数类`0=非确诊`的识别稳定性。

## 2. 数据规模与主要特征

- 原始数据：376 例，35 列。
- 清洗后可训练样本：374 例。
- 标签分布：0 类 22 例，1 类 255 例，2 类 97 例。
- 常规训练/测试划分：299 / 75。
- 医学去交杂器实验因可用变量约束，使用 279 例。

本轮实验主要使用以下几类特征：

1. 激素与 ARR 相关特征：`ARR比值`、`立位醛固酮`、`确诊后_卧位醛固酮醛固酮`及其多时点复测值、`肾素`及其多时点复测值。
2. 一般临床指标：`收缩压`、`舒展压`、`钾`、`钠`、`氯`、`肌酐`、`血红蛋白`、`白细胞`、`血小板`、`年龄`、`性别`。
3. 影像与病史相关特征：`是否有肾上腺结节`、`是否有增生`、`结节最大直径`。
4. 检查与治疗相关特征：`确诊实验类型`、`RASS/利尿剂/Beta/Alpha/钙拮抗剂`等药物等效分数、`联合用药_总数`。
5. 缺失模式特征：对关键数值变量额外构造缺失指示列。

从多轮模型的重要性结果看，最稳定的核心变量集中在`确诊后卧位醛固酮`、`肾素`多时点指标、`ARR比值`、`立位醛固酮`、`钾`、`舒展压`以及若干关键实验室指标的缺失指示项，说明模型主要依赖与 ARR 轴和盐皮质激素相关的生理信号进行判别。

### 2.1 图中特征名释义

结果图中的特征名大多来自预处理后的模型输入，因此会带有统一前缀：

- `num__`：表示该特征以数值变量形式进入模型。
- `cat__`：表示该特征以类别变量形式进入模型。
- `missingindicator_`：表示“该项原始数据是否缺失”的指示变量，取值通常为 0/1。若它的重要性较高，说明“是否缺失”本身也携带诊断信息。

图中常见重点特征可按医学含义理解为：

| 图中特征名 | 含义说明 |
|---|---|
| `确诊后_卧位醛固酮醛固酮` | 确诊相关检查后、卧位状态下测得的醛固酮水平，是判断 ARR 相关异常的重要激素指标。 |
| `确诊后_卧位醛固酮醛固酮.1` | 与上面同类的复测值或另一时间点/批次的卧位醛固酮结果；原始表头保留了 `.1` 后缀。 |
| `确诊后_卧位醛固酮醛固酮.2` | 同样属于卧位醛固酮的另一列复测或附加记录值；`.2` 表示原始 Excel 中的重复命名列。 |
| `立位醛固酮` | 站立位测得的醛固酮水平，用于反映体位变化下的激素反应。 |
| `ARR比值` | 醛固酮/肾素比值，是 ARR 筛查中的核心指标。 |
| `肾素`、`肾素.1`、`肾素.2`、`肾素.3` | 肾素及其不同时间点、不同批次或不同记录列的测量值；后缀 `.1/.2/.3` 通常表示原始数据中的重复列名。 |
| `舒展压` | 舒张压，反映血压状态。 |
| `收缩压` | 收缩压，反映血压状态。 |
| `钾` | 血钾水平，原发性醛固酮增多症相关分析中很关键。 |
| `钠` | 血钠水平。 |
| `氯` | 血氯水平。 |
| `肌酐` | 肾功能相关指标。 |
| `血红蛋白` | 血常规指标之一。 |
| `白细胞` | 炎症或一般血常规状态相关指标。 |
| `血小板` | 血常规指标之一。 |
| `谷草` | 谷草转氨酶（AST），肝功能相关指标。 |
| `谷丙` | 谷丙转氨酶（ALT），肝功能相关指标。 |
| `年龄` | 患者年龄。 |
| `性别` | 患者性别。 |
| `是否有肾上腺结节` | 是否存在肾上腺结节影像学表现。 |
| `是否有增生` | 是否提示肾上腺增生。 |
| `结节最大直径` | 肾上腺结节最大直径，反映影像学病灶大小。 |
| `确诊实验类型` | 采用的确诊试验类型，如冷盐水实验或卡托普利实验。 |
| `二氢吡啶类_等效分数` | 二氢吡啶类降压药使用强度的等效评分。 |
| `RASS_等效分数` | 肾素-血管紧张素-醛固酮系统相关用药强度的等效评分。 |
| `利尿剂_等效分数` | 利尿剂使用强度的等效评分。 |
| `Beta_等效分数` | Beta 受体阻滞剂使用强度的等效评分。 |
| `Alpha_等效分数` | Alpha 受体阻滞剂使用强度的等效评分。 |
| `非二氢吡啶类_等效分数` | 非二氢吡啶类钙拮抗剂使用强度的等效评分。 |
| `联合用药_总数` | 患者当前联合降压或相关药物的总数量。 |
| `ARR比值>192为阳性，推荐进行确诊试验` | 按经验阈值判断 ARR 是否达到阳性筛查标准。 |

整体来看，这些高权重特征可以归纳为四大类：一是 ARR 轴核心激素指标，二是血压与电解质指标，三是影像学和确诊试验信息，四是药物暴露及缺失模式信息。

## 3. 实验方法与对应效果

| 方法 | 说明 | 代表结果 |
|---|---|---:|
| Random Forest 基线 | 最早期三分类基线 | Accuracy 0.7600，Macro F1 0.4795，Balanced Accuracy 0.4840 |
| 升级版集成主线 | 比较 XGBoost、LightGBM、CatBoost、Stacking、两阶段集成等 | 最优为 `main_xgboost`，Accuracy 0.9467，Macro F1 0.8623，Balanced Accuracy 0.8869 |
| 因果增强 XGBoost 变体 | 比较有序损失、EMD、特征加权、DML 去交杂等 | 子实验最优为 `ordinal_emd_fw_gpl_xgboost`，Accuracy 0.9600，Macro F1 0.8935，Balanced Accuracy 0.8935 |
| TabPFN 单独实验 | 与 XGBoost 做公平对比 | Accuracy 0.9733，Macro F1 0.8825，Balanced Accuracy 0.8333 |
| TabPFN + 受控 ADASYN | 扩增到 573 个训练样本 | `tabpfn_adasyn_base`：Accuracy 0.9600，Macro F1 0.8708，Balanced Accuracy 0.8167 |
| 前沿并行实验 | 新增 TAP-inspired 与 CausalMixFT-inspired 代理增广主线，与历史结果并行比较 | 总体最优为 `tabpfn_reference`，少数类最稳为 `xgb_scm_proxy` |
| RAAS-PINN | 加入 RAAS 先验软约束的探索性方案 | Accuracy 0.5467，Macro F1 0.3982，效果明显落后 |
| 医学去交杂器 | 潜因子去交杂与反事实 ARR 探索 | 诊断任务最佳 Accuracy 0.5714，Macro F1 0.3838；更适合机制探索而非最终分类 |

## 4. 结果解读

1. **基线模型能跑通流程，但不足以支撑最终结论。**  
   Random Forest 对少数类`0=非确诊`几乎没有识别能力，说明该任务对类别不平衡非常敏感。

2. **复杂集成不一定优于最优单模型。**  
   在升级版集成实验中，等权投票、全局加权、Stacking、两阶段集成都未稳定超过单模型 XGBoost；最佳传统机器学习方案仍是`main_xgboost`。

3. **TabPFN 在总体性能上表现很强，但少数类稳定性略弱。**  
   TabPFN 在 Accuracy、Macro F1、AUC 上表现突出，说明其对小样本表格判别边界学习能力较强；但在`0=非确诊`召回和 Balanced Accuracy 上，XGBoost 仍更稳。

4. **因果增强 XGBoost 变体显示出一定潜力。**  
   在因果增强子实验中，`ordinal_emd_fw_gpl_xgboost`取得了当前变体组内最佳综合指标，提示将类别有序结构和更平滑的概率目标结合，可能有助于灰区样本表达。

5. **新增的结构因果一致增广代理方案对少数类稳健性有帮助。**  
   在新并行实验中，`xgb_scm_proxy` 在保持 Accuracy `0.9467` 不变的前提下，将 Balanced Accuracy 从 `0.8869` 提升到 `0.8971`，`0=非确诊` 的 recall 保持在 `0.75`，说明按“药物暴露 -> 肾素/ARR -> 确诊后激素”顺序生成训练样本，较单纯局部修补更有利于稳健性。

6. **局部修复式 TAP-inspired 代理增广没有带来稳定增益。**  
   `xgb_tap_proxy` 虽然 AUC 较高，但 Macro F1 和 Balanced Accuracy 均低于基线，提示仅做局部数值修补容易引入与真实临床分布不一致的边界扰动。

7. **机制建模路线目前更适合作为补充研究。**  
   RAAS-PINN 与医学去交杂器在当前样本规模和数据形态下尚未转化为更优的分类性能，但对后续机制解释、反事实分析和论文讨论仍有价值。

## 5. 当前阶段结论

- 如果优先看总体分类性能，当前单次前向结果中 `tabpfn_reference` 仍是并行对比里的最高 `Macro F1` 方案。
- 如果优先看稳健性与少数类表现，新增的 `xgb_scm_proxy` 已经略优于 `main_xgboost / xgb_reference`。
- 如果优先看创新方法展示，可将“因果增强 XGBoost 变体”与“结构因果一致增广代理方案”并列呈现，前者强调目标函数创新，后者强调样本生成机制。
- 本轮 `TabPFN` 新代理增广方案未实际运行，因为运行环境缺少 `TabPFN token`；因此前沿并行实验中的 `TabPFN` 部分仍以历史结果引用为主。
- 因此，当前建议将`TabPFN 单次前向结果`作为总体性能代表，将`xgb_scm_proxy`作为少数类稳健代表，并继续保留`main_xgboost`与`ordinal_emd_fw_gpl_xgboost`作为传统可解释对照。
