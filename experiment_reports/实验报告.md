# 基于 ARR 临床真实世界数据的随机森林与异构集成学习实验报告

## 1. 研究背景与目标

本项目围绕 `ARR` 相关临床真实世界数据，构建并比较多种机器学习模型与集成学习方法，用于辅助预测患者确诊结果。整体工作分为五个阶段：

1. 基于三分类目标的 `Random Forest` 初始基线实验。
2. 基于三分类目标的第一版异构集成实验：`XGBoost + LightGBM + CatBoost` 等权软投票。
3. 基于三分类目标的升级版多策略集成实验：系统比较加权融合、类别感知融合、`Stacking`、两阶段分层集成与中间注入式集成，并结合可解释性分析输出结论。
4. 基于官方 `tabpfn_client` 的 `TabPFN-3` 单独实验。
5. 基于 RAAS 先验软约束的 `RAAS-PINN` 探索性实验。

本报告目的是完整总结已经完成的全部实验过程、参数设置、模型表现、图表结果、可解释性分析以及最终建议，为后续论文撰写、项目汇报或进一步实验迭代提供依据。

## 4. 实验一：三分类 Random Forest 基线

### 4.1 实验目的

构建一个简单、稳定、可快速运行的三分类树模型基线，验证在与后续实验一致的 `0/1/2` 任务定义下，随机森林的可用性与性能上限。

### 4.2 任务定义

- 输入：除目标列以外的所有变量。
- 输出：`0 / 1 / 2` 三分类标签。
- 标签处理方式：
  - 使用全部有效 `0 / 1 / 2` 样本参与训练。
  - 仅剔除目标缺失或非法标签样本。

### 4.3 训练样本情况

根据 `outputs/metrics.json`：

- 总样本：`376`
- 用于训练的 `0/1/2` 样本：`374`
- 被排除的缺失或非法目标样本：`2`
- 训练标签分布：
  - `0`：`22`
  - `1`：`255`
  - `2`：`97`

### 4.4 模型设置

- 模型：`RandomForestClassifier`
- 树数量：`500`
- 测试集比例：`20%`
- 随机种子：`42`

### 4.5 实验结果

根据 `outputs/metrics.json`：

- Accuracy：`0.7600`
- Balanced Accuracy：`0.4840`
- Macro Precision：`0.4801`
- Macro F1：`0.4795`
- Weighted F1：`0.7325`
- OVR ROC AUC Macro：`0.8708`

### 4.6 结果分析

虽然该模型作为最初始三分类基线已经能够识别大多数 `1=确诊` 和部分 `2=灰色区域` 样本，但整体结果明显落后于后续 `XGBoost` 与 `TabPFN-3`。混淆矩阵显示：

- 真实 `0` 类样本 `4` 例全部未被正确识别，模型对 `0=非确诊` 的 recall 仍为 `0`
- `1=确诊` 类 recall 为 `0.9020`
- `2=灰色区域` 类 recall 为 `0.5500`

这说明在当前显著类别不均衡的三分类任务中，随机森林虽可作为统一任务定义下的早期基线，但对少数类和中间态类别的区分能力不足，无法作为最终临床模型。

## 8. 各类集成方式的结果分析

### 8.1 等权软投票

等权软投票是最简单的融合方式，但在当前数据上表现较差：

- 与 `LightGBM` 基本相同
- 明显落后于 `XGBoost`

原因在于三个基模型使用了相似的输入、相似的采样策略和相近的优化目标，错误高度相关，等权平均无法有效利用模型差异。

### 8.2 全局加权融合

全局加权融合在 OOF 上表现很好，但测试集没有带来提升。根据 `strategy_details.json`，学习到的权重大致为：

- `xgboost`: `0.1815`
- `lightgbm`: `0.3605`
- `catboost`: `0.4580`

这说明从训练折看，融合器更偏向 `CatBoost` 和 `LightGBM`，但这种权重模式在测试集未能泛化，最终退化到与等权融合接近的水平。

### 8.3 类别感知加权融合

类别感知加权在 OOF 上表现强于全局加权，也在测试集将 `非确诊` 召回率恢复至 `0.75`，但总体 `Macro F1` 仍未超过 `XGBoost`。其表现追平 `CatBoost`，说明“按类别分配不同模型权重”比全局统一权重更合理，但在当前样本量下仍未形成实质超越。

### 8.4 OOF Stacking

`Stacking` 在 OOF 上极其强势：

- OOF Macro F1：`0.9902`
- OOF AUC：`1.0000`

但测试集只有：

- Macro F1：`0.8482`

说明在当前数据规模下，二层元学习器存在明显过拟合。尽管它确实学习到了哪些模型概率更重要，例如：

- `catboost_prob_1`
- `lightgbm_prob_2`
- `catboost_prob_0`
- `lightgbm_prob_0`

但这种模式没有稳定泛化到独立测试集。

### 8.5 两阶段分层集成

两阶段分层策略利用了标签结构：

1. 先区分 `确诊` 与 `非确诊/灰区`
2. 再在非确诊/灰区内部区分 `0` 与 `2`

该方案在 OOF 上甚至达到完美，但测试集：

- Macro F1：`0.8367`

虽然它保留了 `非确诊` 的召回能力，但对 `确诊` 类召回略有下降，说明阶段间误差传递在小样本场景下更加明显。

### 8.6 中间注入式集成

中间注入实验采用的是树模型可实现版本：

- `XGBoost` 叶子索引
- `LightGBM` 叶子索引
- `CatBoost` raw margin
- 三模型概率输出
- 少量关键原始临床特征

虽然在 OOF 上达到完美，但测试集：

- Balanced Accuracy：`0.8101`
- Macro F1：`0.8151`

明显不如最佳单模型，说明“中间表示”进一步增加了模型自由度，在当前小样本条件下更容易过拟合训练集结构。

## 9. 概率校准与阈值优化结果

对最优策略 `main_xgboost` 进一步做了：

- `sigmoid` 校准
- `isotonic` 校准
- 校准后类别阈值缩放

结果见 `ensemble_outputs_v2/tables/best_strategy_calibration_results.csv`：

| 方法 | Accuracy | Balanced Accuracy | Macro F1 |
|---|---:|---:|---:|
| default | 0.9467 | 0.8869 | 0.8623 |
| sigmoid | 0.9333 | 0.8703 | 0.8210 |
| sigmoid_thresholded | 0.9333 | 0.8703 | 0.8210 |
| isotonic | 0.9467 | 0.8869 | 0.8623 |
| isotonic_thresholded | 0.9467 | 0.8869 | 0.8623 |

结论：

- `sigmoid` 校准反而导致性能下降。
- `isotonic` 与默认结果持平。
- 阈值缩放未带来额外收益。

因此最终保留默认 `XGBoost` 输出作为最优版本。

说明模型在个体级别上主要依据多时点醛固酮与肾素相关信息进行判定，这与疾病生理机制具有一致性。

### 12.5 TabPFN-3 结果解释

因此，在“TabPFN-3 单独实验”这一轮比较中，`TabPFN-3` 可视为当轮总体性能更优的模型；但在后续加入代价敏感学习与受控 `ADASYN` 的扩展实验后，最终应以扩展实验结论为准。

3. `TabPFN-3` 经过受控 `ADASYN` 扩增后，基础版性能可进一步提升，得到新的总体更优版本：`tabpfn_adasyn_base`。
4. `TabPFN-3 think-mode` 在当前扩增数据上并不适用，反而明显退化，不建议作为本项目正式结果版本。

- 保留教师模型筛选，避免明显偏离目标类的合成样本进入训练

## 15. 综合结论

结合全部实验，得到以下结论：

1. 三分类随机森林虽然可作为流程基线，但对少数类 `0` 仍基本无识别能力，且对 `2=灰色区域` 的识别也明显有限，不适合作为最终模型。
2. 在三分类任务中，简单等权软投票并不能优于最佳单模型，说明“有多个模型”不等于“集成一定更好”。
3. 在树模型与集成模型内部比较中，`XGBoost` 仍是最优的传统机器学习方案。
4. `TabPFN-3` 单独实验在总体指标上超过了 `XGBoost`，是当前全部实验中 `Macro F1` 和 `AUC` 最优的模型。
5. `XGBoost` 的 `Balanced Accuracy` 和 `0=非确诊` 召回率仍优于 `TabPFN-3`，因此在少数类更敏感的临床情境下仍有保守价值。
6. 在代价敏感学习与受控 `ADASYN` 扩展实验中，`TabPFN-3` 的最佳版本已经从原始基础版变为 `tabpfn_adasyn_base`，其总体指标优于 `XGBoost`。
7. `TabPFN-3` 的 `think-mode` 在扩增后的数据上明显退化，说明测试时/拟合时额外计算并不能自动抵消合成采样带来的噪声。
8. `RAAS-PINN` 在当前数据上未体现优势，说明机制约束模型对真实时间轴与药代参数完整性依赖更强。
9. 在新增前沿并行实验中，`xgb_scm_proxy` 已经在少数类稳健性上略优于 `xgb_reference`，说明结构因果一致增广是值得继续推进的新方向。
10. 从可解释性角度看，树模型最稳定的判定依据仍集中在多时点醛固酮、肾素及其缺失模式上，具有一定临床合理性。

## 16. 局限性与后续建议

### 16.1 局限性

1. 样本量有限，仅 `374` 例。
2. `0=非确诊` 类样本只有 `22` 例，极易造成模型不稳定。
3. 当前仍为单中心/单表格级别验证，缺乏外部中心验证。
4. 集成模型在 OOF 上优异但测试集下降，提示高复杂度结构对该数据规模并不友好。
5. 当前 `TabPFN` 新增广代理版本未实际跑通，主要受限于运行环境中缺少 token，而不是方法本身已被证伪。

### 16.2 后续建议

1. 如果优先目标是总体性能，建议优先采用 `tabpfn_adasyn_base`。
2. 如果优先目标是提升 `0=非确诊` 识别稳定性，可继续保留 `XGBoost` 或 `tabpfn_base_original` 作为对照或备选模型。
3. 如果下一步目标是进一步提升 `0` 类识别能力，建议优先：
   - 做更严格的外部中心验证或留中心验证；
   - 引入更有区分力的临床特征；
   - 对 `TabPFN-3` 与 `XGBoost` 做针对少数类的定向对比或后融合；
   - 尝试更强的代价敏感损失，而非继续增加集成复杂度。
4. 在论文或汇报中，可将复杂集成实验作为“方法比较部分”，并据此强调：
   - 在小样本临床数据中，复杂集成不一定优于最佳单模型；
   - 过度复杂的融合结构可能引入额外过拟合风险；
   - 模型选择应兼顾总体性能、少数类稳定性与可解释性。
5. 如果后续继续做前沿探索，建议优先保留 `xgb_scm_proxy` 方向，并把 `TabLLM / Schema-Adaptive Learning` 作为下一阶段语义增强路线。

## 17. 最终建议模型

当前项目的最终建议分为三个层次：

- 总体性能最优模型：`tabpfn_adasyn_base`
- 少数类更稳的保守模型：`XGBoost` 单模型三分类方案
- 原始小样本更稳的 TabPFN 备选：`tabpfn_base_original`

推荐理由：

1. `tabpfn_adasyn_base` 在当前全部已完成实验中取得了更高的 `Accuracy`、`Macro F1` 和 `Weighted F1`，是新的总体性能最优版本。
2. `XGBoost` 的 `Balanced Accuracy` 与 `0=非确诊` 召回率仍更稳，适合作为临床更保守的辅助对照模型。
3. `tabpfn_base_original` 在不进行扩增时具有更高的 `Balanced Accuracy`，适合作为原始小样本条件下的 TabPFN 参考版本。
4. `TabPFN-3 think-mode` 在扩增数据上表现明显退化，不建议作为正式结果版本。
5. `RAAS-PINN` 结果明显偏弱，目前仅适合作为机制建模探索，不适合作为最终模型。
6. 因此，若用于当前数据的主结果展示，推荐以 `tabpfn_adasyn_base` 为主、`XGBoost` 为辅，并保留 `tabpfn_base_original` 作为额外参考。
