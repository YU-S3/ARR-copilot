# 前沿并行实验专项报告

## 1. 实验目的

在当前 `ARR` 临床真实世界表格数据上，新增一条与既有主线并行的“前沿方法对比”实验，验证以下问题：

1. 在当前小样本医疗表格任务中，`Tabular Foundation Models` 的单次前向推理能力是否仍然优于传统树模型。
2. 如果不直接微调大模型，而是围绕小样本表格做结构化增广，哪一类增广更有价值：
   - `TAP-inspired` 的局部修复式增广；
   - `CausalMixFT-inspired` 的结构因果一致增广。
3. 在“总体性能”和“少数类稳健性”这两个目标并行汇报时，是否会出现不同的最佳方案。

本次实验保持与既有实验相同的数据切分和评估口径，与原始 `XGBoost`、既有 `TabPFN` 结果直接公平比较。

### 3.4 `xgb_tap_proxy`：局部修复式增广代理方案

技术动机：

- 传统 `ADASYN/Mixup` 更像插值式扩增，但医疗表格往往存在强语义边界；
- 某些变量如年龄、性别、药物暴露、影像学结节信息相对稳定，而激素、电解质、血压等更像“可局部修复”的状态变量。

核心思想：

- 锚定一部分较稳定特征不变；
- 仅在同类样本的局部邻域内，对激素和生理指标做有限幅度采样；
- 再用教师模型过滤掉低效用候选样本。

特征分组：

- **锚定特征**：年龄、性别、确诊实验类型、影像学变量、药物等效分数、联合用药数
- **修复特征**：`ARR比值`、醛固酮、肾素、血压、电解质、肝肾功能、血常规指标

生成流程：

1. 在同类样本中找近邻；
2. 对修复特征按邻域均值和方差做截断采样；
3. 对离散变量做四舍五入并裁剪到支持集；
4. 用教师 `XGBoost` 评估目标类置信度；
5. 仅保留高置信候选样本。

工程实现：

- 关键模块：[frontier_augmentation.py](file:///c:/Users/YCY/Desktop/ARR-model/frontier_augmentation.py) 中的 `TapInspiredInpaintingAugmentor`

数学上可近似理解为：

- 给定同类种子样本 $x_i$ 与其邻域 $\mathcal{N}(x_i)$；
- 对允许修复的变量集合 $S$ 中每一维 $j$，生成：
- $\tilde{x}_{ij} \sim \text{TruncatedNormal}(\mu_j(\mathcal{N}), \sigma_j^2(\mathcal{N}))$
- 再通过教师模型 $f_{\text{teacher}}$ 计算：
- $u(\tilde{x}_i) = P_{\text{teacher}}(y=c \mid \tilde{x}_i)$
- 仅保留满足 $u(\tilde{x}_i)$ 足够高的候选样本。

方法价值：

- 有明确的局部修复叙事；
- 工程上容易实现。

方法局限：

- 它没有显式建模“药物变化是否应联动影响 ARR 与激素”；
- 因而更容易引入局部合理、全局不一致的训练样本。

### 3.5 `xgb_scm_proxy`：结构因果一致增广代理方案

技术动机：

- 当前数据中药物等效分数、肾素、ARR、确诊后激素之间存在明显因果耦合；
- 若只做局部修补而不考虑“先改药物、再联动改 ARR/激素”的方向，生成样本可能破坏临床逻辑。

核心思想：

- 不做完全数据驱动的因果图发现；
- 直接用医学先验指定变量顺序，再对每个下游节点拟合轻量模型；
- 生成时按顺序采样，保持结构一致性。

结构顺序：

1. 外生变量：年龄、性别、影像学变量
2. 治疗变量：各类药物等效分数、联合用药数
3. 初筛生理变量：肾素、立位醛固酮、`ARR比值`、血压、电解质
4. 确诊后变量：卧位醛固酮多时点值、肾素复测值

生成流程：

1. 先固定外生变量；
2. 对治疗变量做轻度干预或重采样；
3. 按结构顺序用轻量 `XGBRegressor` 生成下游变量；
4. 用残差自助法保留一定随机性；
5. 再用教师模型过滤掉低置信候选样本。

工程实现：

- 关键模块：[frontier_augmentation.py](file:///c:/Users/YCY/Desktop/ARR-model/frontier_augmentation.py) 中的 `SCMMixAugmentor`

数学上可理解为：

- 将变量按拓扑顺序排列为：
- $Z \rightarrow T \rightarrow H \rightarrow P$
- 其中：
  - $Z$：外生变量
  - $T$：治疗变量
  - $H$：初筛生理变量
  - $P$：确诊后变量
- 对每个节点学习条件模型：
- $X_j = g_j(\text{Pa}(X_j)) + \epsilon_j$
- 采样时按顺序递推生成：
- $\tilde{X}_j = \hat{g}_j(\widetilde{\text{Pa}(X_j)}) + \tilde{\epsilon}_j$

方法价值：

- 能把“药物变化 -> 肾素/ARR变化 -> 确诊后激素变化”的链条编码进合成样本；
- 比局部修复式增广更贴近临床逻辑。

方法局限：

- 仍然是代理实现，不是完整论文原版复现；
- 结构顺序主要来自医学先验，而不是严格学习得到。

### 5.2 总体最优方案

按当前项目“总体性能优先”的主评价口径：

- `Macro F1`
- `Accuracy`
- `OVR ROC AUC`

综合判断，**当前并行实验中的总体最优方案仍是 `tabpfn_reference`**。

其测试集结果为：

- Accuracy：`0.9733`
- Balanced Accuracy：`0.8333`
- Macro F1：`0.8825`
- Weighted F1：`0.9691`
- OVR ROC AUC Macro：`0.9939`

与最稳传统基线 `xgb_reference` 相比：

- Accuracy：`0.9467 -> 0.9733`
- Macro F1：`0.8623 -> 0.8825`
- AUC：`0.9702 -> 0.9939`

这说明在当前小样本任务上，表格基础模型在总体判别能力上依然保持明显优势。

### 5.3 少数类稳健最优方案

按当前项目“少数类稳健性优先”的评价口径：

- `0类 recall`
- `Balanced Accuracy`
- `Macro F1`

综合判断，**当前少数类最稳健的新方案已经从 `xgb_reference` 更新为 `xgb_scm_proxy`**。

其测试集结果为：

- Accuracy：`0.9467`
- Balanced Accuracy：`0.8971`
- Macro F1：`0.8674`
- `0类 recall`：`0.75`

与 `xgb_reference` 相比：

- Accuracy：`0.9467 -> 0.9467`
- Balanced Accuracy：`0.8869 -> 0.8971`
- Macro F1：`0.8623 -> 0.8674`

这说明结构因果一致增广在不牺牲整体准确率的前提下，确实带来了更稳的平衡表现。

### 5.4 当前前沿主线的分层结论

如果按当前结果分成三档：

- **第一档：`tabpfn_reference`**
- **第二档：`tabpfn_adasyn_reference` 与 `xgb_scm_proxy`**
- **第三档：`xgb_reference` 与 `xgb_tap_proxy`**

其中：

- `tabpfn_reference`
  - Overall 指标最强
  - `0类 recall = 0.50`

- `xgb_scm_proxy`
  - 当前少数类稳健最优
  - `0类 recall = 0.75`
  - `Balanced Accuracy = 0.8971`

- `xgb_tap_proxy`
  - AUC 很高，但宏平均与平衡准确率明显下降

结论：

- 如果目标是“当前数据上的绝对分类性能”，主推 `tabpfn_reference`
- 如果目标是“少数类稳健 + 传统树模型可控性”，主推 `xgb_scm_proxy`
- 如果目标是“局部修复式增广是否可行”的方法学探索，`xgb_tap_proxy` 已经给出偏负面的证据

## 6. 各方案结果解读

### 6.1 `xgb_reference`

结果：

- Accuracy：`0.9467`
- Balanced Accuracy：`0.8869`
- Macro F1：`0.8623`
- `0类 recall`：`0.75`

解读：

- 它仍然是当前最稳的传统树模型基线；
- 在少数类上比 `TabPFN` 更保守、更稳。

结论：

- 仍然适合作为所有前沿路线的参照模型；
- 但已经不再是当前“少数类稳健性”的唯一最佳方案。

### 6.2 `tabpfn_reference`

结果：

- Accuracy：`0.9733`
- Balanced Accuracy：`0.8333`
- Macro F1：`0.8825`
- `0类 recall`：`0.50`

解读：

- 总体性能极强；
- 对 `确诊` 与 `灰色区域` 的识别几乎接近上限；
- 但 `0类 recall` 仍明显弱于树模型主线。

结论：

- 仍是当前“总体性能最优”的单次前向方案；
- 但若临床上更关心排除类稳健性，仍需保留树模型作为对照。

### 6.3 `tabpfn_adasyn_reference`

结果：

- Accuracy：`0.9600`
- Balanced Accuracy：`0.8167`
- Macro F1：`0.8708`
- `0类 recall`：`0.50`

解读：

- 相比原始 `tabpfn_reference`，总体性能仍强；
- 但少数类稳健性没有同步提升。

结论：

- 受控扩增对 `TabPFN` 的收益主要体现在总体判别边界；
- 不足以解决 `0类` 识别偏弱的问题。

### 6.4 `xgb_tap_proxy`

结果：

- Accuracy：`0.9467`
- Balanced Accuracy：`0.8203`
- Macro F1：`0.8203`
- `0类 recall`：`0.50`

解读：

- 虽然 AUC 达到 `0.9916`，但离散分类效果明显下降；
- 说明这类局部修复式增广更像是在重塑排序边界，而不是增强稳定分类决策；
- 它把 `0类 recall` 从 `0.75` 拉低到 `0.50`，这是当前最不希望看到的退化。

结论：

- 当前代理实现下，`TAP-inspired` 路线不适合作为主线；
- 可保留为“负结果”或方法学探索补充。

### 6.5 `xgb_scm_proxy`

结果：

- Accuracy：`0.9467`
- Balanced Accuracy：`0.8971`
- Macro F1：`0.8674`
- `0类 recall`：`0.75`

解读：

- 在不损失 Accuracy 的情况下，小幅提升了 Balanced Accuracy 与 Macro F1；
- `0类 recall` 保持稳定，说明其收益不是靠牺牲少数类换来的；
- `灰色区域` 的 recall 达到 `1.0`，提示结构一致增广可能有助于灰区边界表达。

结论：

- 这是本轮新增方法中最值得保留的正向结果；
- 它证明“因果一致的样本生成”比“局部修补式样本生成”更适合当前医疗表格任务。

## 7. 增广审计结果

### 7.1 审计总原则

根据 [augmentation_audit.csv](file:///c:/Users/YCY/Desktop/ARR-model/frontier_parallel_outputs/tables/augmentation_audit.csv)，本轮对所有生成样本额外记录：

- 各类扩增数量
- 教师模型目标类置信度
- 与种子样本的局部距离
- 特征均值偏移
- 特征标准差偏移
- 是否超出训练集支持范围

### 7.2 `tap_proxy` 审计结论

关键现象：

- 新增 `0类` 样本：`72`
- 新增 `2类` 样本：`117`
- 平均教师目标类概率：`0.8355`
- `out_of_support_rate` 全部为 `0`

解读：

- 从“是否越界”角度看，生成样本没有明显非法值；
- 但从分布漂移角度看，多个核心激素变量偏移较大，例如：
  - `ARR比值`
  - `立位醛固酮`
  - `确诊后_卧位醛固酮醛固酮`
  - `确诊后_卧位醛固酮醛固酮.1`

这说明：

- 即使局部采样不产生越界值，也可能在多变量联合结构上偏离真实临床流形；
- 这与其最终分类性能下降是一致的。

### 7.3 `scm_proxy` 审计结论

关键现象：

- 新增 `0类` 样本：`72`
- 平均教师目标类概率：`0.7097`
- 平均与种子距离：`0.1099`
- `out_of_support_rate` 全部为 `0`

解读：

- `scm_proxy` 的教师置信度略低于 `tap_proxy`，说明它生成的样本没有那么“极端像原类”；
- 但其最终测试性能反而更稳定，提示“结构合理但略多样化”的样本，比“局部很像但全局不一致”的样本更有价值；
- 这也是结构因果一致增广的核心优势所在。

### 7.4 本节结论

本轮审计给出两个重要结论：

1. “不越界”不等于“有帮助”，局部修复式增广虽然合法，但不一定保留真实医学结构。
2. 结构一致性比局部相似性更重要，这解释了为什么 `xgb_scm_proxy` 优于 `xgb_tap_proxy`。

## 9. 后续建议

下一步最值得做的是：

1. 保留 `xgb_scm_proxy` 作为新增主线，继续优化结构因果一致增广。
2. 不建议继续把当前 `tap_proxy` 作为正式主路线，可作为对照或负结果保留。
3. 如果后续补齐 `TabPFN token`，优先补跑：
   - `tabpfn_tap_proxy`
   - `tabpfn_scm_proxy`
4. 将 `Schema-Adaptive / TabLLM` 路线作为下一阶段“表格转语义表征”的扩展方向，而不是本轮主线。
5. 在论文或汇报中，可以明确区分：
   - **总体性能最优**：`tabpfn_reference`
   - **少数类稳健最优**：`xgb_scm_proxy`
   - **传统稳健基线**：`xgb_reference`
