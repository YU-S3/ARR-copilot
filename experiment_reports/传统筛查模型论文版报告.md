# 基于 data_0428.xlsx 的传统筛查模型实验报告

日期：2026-06-06

## 摘要

本报告汇总 `data_0428.xlsx` 更新后围绕“传统筛查模型”完成的主要实验。研究目标是在不使用确诊试验后信息的前提下，构建可用于原醛相关筛查的表格机器学习模型，并比较三分类诊断任务与二分类筛查任务的稳定性、过拟合风险和临床可用阈值。

新表有效标签分布为 `0=37, 1=210, 2=128`。当前正式筛查特征集定义为删除 `试验后醛固酮` 与 `试验后肾素` 两列，其余可用筛查、实验前、药物、影像和基础生化变量保留。数据管线禁止使用住院号、索引列和 missing indicator 特征；数值缺失采用中位数填补，药物等效分数缺失按 0 处理。

主要结论如下：

1. 全特征模型性能接近满分，但强依赖 `试验后醛固酮/试验后肾素`。遮蔽这两项后，三分类 CatBoost BA 从 `0.997` 降至 `0.333`，二分类 CatBoost BA 从 `0.999` 降至 `0.563`。因此全特征模型仅可作为信息上限，不适合作为筛查模型。
2. 正式筛查特征集下，二分类任务更适合筛查。10 折 CV 中普通 XGBoost 二分类达到 BA `0.783`、sensitivity `0.959`、specificity `0.608`，优于 SCM 二分类的 BA `0.752`。
3. 三分类任务在正式筛查特征下存在明显难度。10 折 CV 最优为 CatBoost，BA `0.705`、macro F1 `0.708`，class 0 recall `0.592`，class 2 recall `0.672`。
4. 正则化和阈值校准实验显示，二分类正则化 XGBoost 是当前最稳健的传统筛查候选。`xgb_bin_d3_l20_bal + sigmoid + sensitivity_0_90_spec_max` 在 confirmation 10 折 CV 中达到 BA `0.761`、accuracy `0.900`、sensitivity `0.934`、specificity `0.588`，BA gap `0.070`。
5. 如果强制 sensitivity 更高，整体准确率与 specificity 会下降。`sensitivity >= 0.93` 时最佳约为 BA `0.747`、specificity `0.545`；`sensitivity >= 0.95` 时最佳约为 BA `0.724`、specificity `0.490`。这反映筛查模型中“少漏诊”和“少误报”的固有权衡。
6. SCM 增广和因果/ordinal 变体在当前传统筛查任务中没有形成稳定优势。后续优先方向应是传统模型的医学派生特征、约束阈值优化、soft-voting/传统集成和错误样本分析，而不是继续扩大 SCM 矩阵。
7. 后续完整约束实验已完成。医学派生特征与传统集成没有显著突破原正则化 XGBoost 的 sensitivity-specificity 前沿：`sensitivity >= 0.93` 时 `xgb_eng_d2_l10 + sigmoid` 达到 BA `0.741`、sensitivity `0.949`、specificity `0.534`；`sensitivity >= 0.95` 时 `soft_vote_xgb_cat + sigmoid` 达到 BA `0.711`、sensitivity `0.960`、specificity `0.462`。因此当前不建议继续扩大传统集成矩阵，应转向错误样本和数据层面改进。

```text

```

```text

0 = 非确诊
1 = 确诊或灰色区域
```

二分类任务更符合筛查逻辑，即重点识别“需要进一步关注或处理”的患者。

## 3. 方法与公式

### 3.1 缺失值处理

数值变量采用训练折中位数填补：

```text
x_ij' = median_j(X_train), if x_ij is missing
x_ij' = x_ij, otherwise
```

分类变量采用常量 `"Missing"` 填补后编码。药物等效分数列缺失按 0 填补，表示未记录用药贡献。

特别地，数值填补中设置：

```text
add_indicator = False
```

因此不会生成 `missingindicator_*` 特征，避免模型将“是否缺失”作为医学指标。

### 3.2 XGBoost

XGBoost 使用加性树模型：

```text
F_K(x) = sum_{k=1}^K f_k(x)
```

二分类概率为：

```text
p(y=1|x) = sigmoid(F_K(x)) = 1 / (1 + exp(-F_K(x)))
```

训练目标为：

```text
L = sum_i l(y_i, F_K(x_i)) + sum_k Omega(f_k)
```

其中正则项为：

```text
Omega(f) = gamma * T + 1/2 * lambda * sum_j w_j^2 + alpha * sum_j |w_j|
```

`T` 为叶节点数，`w_j` 为叶节点权重。正则化实验主要通过降低 `max_depth`、提高 `min_child_weight`、`gamma`、`reg_lambda` 和 `reg_alpha` 来降低过拟合。

### 3.3 CatBoost

CatBoost 使用梯度提升决策树，并通过 ordered boosting 和对称树结构降低类别特征编码泄漏和过拟合风险。其基本形式同样为：

```text
F_K(x) = sum_{k=1}^K eta * f_k(x)
```

其中 `eta` 为学习率。实验中通过 `depth` 和 `l2_leaf_reg` 控制模型复杂度。

### 3.4 ADASYN 增广

ADASYN 根据少数类样本邻域难度生成合成样本。抽象形式为：

```text
x_new = x_i + lambda * (x_nn - x_i), lambda ~ Uniform(0, 1)
```

其中 `x_i` 为少数类样本，`x_nn` 为其邻近样本。项目中对离散数值变量进行了四舍五入和范围裁剪，以减少不合理合成值。

### 3.5 SCM-v2 结构化增广

SCM-v2 不是无约束插值，而是根据医学结构将变量划分为筛查相关、试验相关、用药相关、基础生化/影像等组别，并在可解释分组内进行有约束混合与扰动。可抽象为：

```text
X_aug = g(X_seed, X_donor, M_group, epsilon)
```

其中 `M_group` 为变量组别掩码，`epsilon` 为残差扰动。其设计目标是增强 class 0 与 class 2 等困难类别，但实验结果显示，在正式筛查特征集下，SCM 目前没有稳定优于普通正则化 XGBoost 或 CatBoost。

### 3.6 Ordinal 三分类

Ordinal 方法将三分类转换为两个二分类边界：

```text
q0(x) = P(y > 0 | x)
q1(x) = P(y > 1 | x)
```

并恢复类别概率：

```text
P(y=0|x) = 1 - q0(x)
P(y=1|x) = q1(x)
P(y=2|x) = q0(x) - q1(x)
```

该方法试图利用非确诊、灰区、确诊之间的序关系，但在当前数据中并未稳定超过 CatBoost。

### 3.7 概率校准

Platt sigmoid 校准：

```text
p_cal = sigmoid(a * logit(p) + b)
```

Isotonic 校准：

```text
p_cal = h(p), where h is monotonic
```

由于样本量较小，isotonic 可能降低 Brier/ECE，但也可能使固定 0.5 阈值失去直观意义。因此最终报告同时比较未校准、sigmoid 和 isotonic。

### 3.8 约束阈值选择

筛查任务中，阈值不应只最大化 accuracy。对给定 sensitivity 下限 `c`，阈值选择定义为：

```text
tau* = argmax_tau Specificity(tau)
subject to Sensitivity(tau) >= c
```

或：

```text
tau* = argmax_tau Accuracy(tau)
subject to Sensitivity(tau) >= c
```

本项目重点比较 `c in {0.90, 0.93, 0.95}`。

### 3.9 评价指标

二分类混淆矩阵定义：

| | 预测阳性 | 预测阴性 |
|---|---:|---:|
| 真实阳性 | TP | FN |
| 真实阴性 | FP | TN |

Sensitivity：

```text
Sensitivity = TP / (TP + FN)
```

Specificity：

```text
Specificity = TN / (TN + FP)
```

Accuracy：

```text
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

二分类 balanced accuracy：

```text
BA = (Sensitivity + Specificity) / 2
```

三分类 balanced accuracy：

```text
BA = (Recall_0 + Recall_1 + Recall_2) / 3
```

F1：

```text
F1 = 2 * Precision * Recall / (Precision + Recall)
```

Macro F1：

```text
Macro F1 = mean(F1_0, F1_1, ..., F1_C)
```

Brier score：

```text
Brier = mean_i sum_c (I(y_i=c) - p_ic)^2
```

ECE：

```text
ECE = sum_b |acc(B_b) - conf(B_b)| * |B_b| / n
```

Train-valid gap：

```text
Gap_metric = Metric_train - Metric_valid
```

若 BA 或 macro F1 gap 较大，说明模型对训练数据记忆明显，泛化风险较高。

## 6. 讨论

### 6.1 为什么全特征模型不能用于筛查

全特征模型接近满分，但遮蔽试验后两项后性能大幅崩塌。这说明模型学习到了确诊试验后的强后验信息，而不是单纯筛查阶段可用信息。若将其用于筛查，会造成临床流程上的信息泄漏。

### 6.2 为什么二分类优于三分类作为筛查主任务

三分类要求模型同时区分非确诊、确诊和灰区。当前数据中 class 0 样本较少，灰区与确诊之间又存在临床边界模糊，因此三分类 BA 稳定在约 `0.70`。二分类将确诊与灰区合并为阳性，更符合“是否需要进一步关注”的筛查目标，也能更清晰地优化 sensitivity。

### 6.3 为什么 sensitivity 约束会降低 BA 或 specificity

在二分类中：

```text
BA = (Sensitivity + Specificity) / 2
```

提高 sensitivity 通常需要降低阳性判定阈值，使更多患者被判为阳性。这会减少 FN，但增加 FP，因此 specificity 下降。当前数据中，当 sensitivity 从约 `0.88` 提高到约 `0.95` 时，specificity 从约 `0.70` 降至约 `0.49`。

### 6.4 SCM 增广为什么暂不作为主线

SCM 在 holdout seed 中偶尔能提高 BA 或 class 2 recall，但在 10 折 CV 和正则化 confirmation 中没有稳定超过普通 XGBoost/CatBoost，且部分设置下 train-valid gap 更大。因此当前阶段不建议继续扩展 SCM 矩阵，除非后续错误样本分析显示存在明确的可建模结构。

### 6.5 当前推荐候选

若以传统筛查模型为主，推荐候选如下：

1. 二分类主模型：

```text
xgb_bin_d3_l20_bal + sigmoid calibration + sensitivity_0_90_spec_max
```

适用场景：希望 sensitivity 保持在 0.90 以上，同时尽量提高 specificity 和 accuracy。该模型在前一轮 confirmation 10 折 CV 中 BA 和 specificity 仍优于本轮医学派生特征/传统集成，因此继续作为第一候选。

2. 更保守筛查阈值：

```text
xgb_bin_d3_l20_bal + sigmoid calibration + sensitivity_0_93_spec_max
```

适用场景：更强调少漏诊，接受更多假阳性。本轮 `xgb_eng_d2_l10 + sigmoid + sens093_maxspec` 可作为补充候选，accuracy 为 `0.908`、BA 为 `0.741`、sensitivity 为 `0.949`、specificity 为 `0.534`，但其 BA gap 高于原主模型，暂不建议替代。

3. 极高 sensitivity 候选：

```text

```

适用场景：若临床流程要求 sensitivity 接近 0.95 或更高，可用该模型作为敏感度优先方案。其 sensitivity 为 `0.960`、accuracy 为 `0.910`，但 specificity 仅 `0.462`；因此必须在报告中明确假阳性代价。

4. 三分类辅助模型：

```text
three_catboost_d3_l50_bal
```

适用场景：需要给出非确诊、确诊、灰区的辅助分层解释。

## 7. 局限性

1. class 0 样本仅 37 例，specificity 和 class 0 recall 方差较大。
2. 当前结果主要来自交叉验证，尚缺外部验证集。
3. `确诊实验类型` 与 `试验前醛固酮/肾素` 被保留，是因为当前正式定义允许实验前信息；若临床筛查场景进一步提前，则需要重新定义更严格特征集。
4. 阈值选择依赖校准折，在小样本下可能有波动，最终应用前需要 bootstrap 或 nested CV 验证阈值稳定性。
5. `screening_constrained_0428` 已完成 full，但仍属于内部交叉验证结果，尚缺外部验证集和 bootstrap/nested CV 对阈值稳定性的进一步确认。

## 9. 结论

基于 `data_0428.xlsx` 的传统筛查模型实验表明，删除试验后两项后，模型性能从接近满分回落到更真实的筛查水平。当前最稳健路线不是继续依赖复杂增广，而是以正则化 XGBoost/CatBoost 为主，结合概率校准和 sensitivity 约束阈值。二分类筛查任务比三分类任务更适合作为主模型；三分类 CatBoost 可作为辅助分层模型。完整约束实验显示，医学派生特征和传统集成没有显著突破原正则化 XGBoost 的 sensitivity-specificity 前沿：在保证较高 sensitivity 时，specificity 仍是主要瓶颈。因此后续重点应从“扩大模型矩阵”转向错误样本分层、阈值稳定性验证、外部验证和数据侧特征质量改进。
